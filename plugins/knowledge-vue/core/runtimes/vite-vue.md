# Runtime: vite-vue

Applies when the active project's runtime is a hand-authored Vite + Vue app (no
meta-framework). This doc owns nothing itself — it declares the branch active and
delegates to the docs and skills that do own the pieces.

- [invariant · desired] Under `runtime = vite-vue`, the app bootstrap is
  **hand-authored** — there is a real `createApp` call, real entry files, and a real
  router instance wired by the project, not generated by a framework. Load the active
  architecture doc (`core/architectures/<a>.md` — path resolution + what-goes-where)
  and the active project-type doc (`core/project-types/<t>.md` — createApp + bootstrap
  process) before placing or writing bootstrap code.
  - ✅ do: resolve `{app}` / `{initial-plugins}` / `{plugins}` via the architecture doc,
    then follow the project-type doc's bootstrap sequence.
  - ❌ don't: guess a bootstrap shape or a directory layout without loading both docs
    first — why: this doc intentionally carries no path resolution or bootstrap steps
    of its own, so guessing duplicates (and risks contradicting) the owning docs.

- [invariant · desired] The routing skills (`pages`, `layouts`, `vue-router`,
  `page-middlewares`) and `plugin-registration` all apply under this runtime — none of
  them are gated off. Use them by name as the corresponding work comes up (adding a
  route, a layout, setting up the router, writing a middleware, wiring a plugin).

This doc's only job is the branch check above: confirm `runtime = vite-vue`, then hand
off to architecture + project-type + the relevant skill. It does not restate any of
their rules.
